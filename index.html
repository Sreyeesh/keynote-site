Technical Workflows

Eight Essential Systems

The backbone of game production

📚

1. Source & Version Control

The first building block. Records every change, who made it, and when.

💾

Purpose

Single source of truth for all changes. Enables branching, merging, and recovery.

🛠️

Tools

Git with Git LFS, Perforce (AAA), Plastic SCM (Unity)

📋

Practices

Branching strategy, commit messages that capture intent, tag builds

Version Control Deep Dive

Branching Strategies

  • Trunk-based for frequent merges
  • Feature-branching for isolation
  • Choose based on team size

Commit Messages

  • Explain *why* not just *what*
  • Turn history into documentation
  • Enable better debugging

2. Build Pipelines

The machinery that transforms source and assets into playable builds

Manual Builds

Fragile, single point of failure

Automated Builds

Reliable, every change validated quickly

Team Confidence

"If it built on the server, it will run on my machine"

Build Pipeline Tools & Practices

Tools

  • Unity Cloud Build
  • Unreal Automation Tool
  • Jenkins, GitHub Actions
  • Azure Pipelines

Best Practices

  • Trigger on commits/merges
  • Integrate smoke tests
  • Archive artifacts
  • Enable rollback

3. Asset Management

Games are asset-heavy. 3D models, textures, animations, sounds, UI, shaders.

Large & Numerous

Without consistent naming, multiple versions creep in

Inconsistent Properties

Scale, pivots, materials create endless bugs

Repository Impact

Large assets can break repositories or balloon builds

Asset Management Solutions

Naming Conventions

CHAR_Hero_Idle_v001.fbx
ENV_Forest_Tree_Oak_01.fbx
UI_MainMenu_Button_Play.png

Folder Structure

Assets/
├── Characters/
├── Environments/
├── Audio/
├── VFX/
└── UI/

Engine Integration

Unity: AssetPostprocessor, Import Presets

Unreal: Reimport rules, Derived Data Cache

4. Testing Workflows

Games are complex systems where one small change can break ten other things

Unit Tests

Check small, isolated systems

Integration Tests

Validate subsystems working together

Smoke Tests

Confirm game launches and loads levels

Performance Tests

Measure FPS, memory, load times

5. Documentation & Communication

Workflows only matter if they are known and followed

Why They Matter

Without documentation, knowledge is siloed and disappears when people leave

Tools

Notion, Confluence, HacknPlan, Slack, Discord

Practices

Document *why* as well as *how*. Connect repos to communication platforms

6. Integration & Release Workflow

Release is when workflows are tested most. Without strong workflows, it's chaos.

Feature Flags

Incomplete features can be merged without breaking the build

Release Checklists

Freeze, validate, package, distribute

Daily Integration

Reduce merge conflicts with regular builds

Rollback Strategies

Archive builds and configs for quick recovery

7. Scope & Vertical Slice Workflow

Scope kills projects more than anything else. That's why we manage it with vertical slices.

Why It Matters

Ambition without discipline leads to unfinished projects

Vertical Slice

One complete piece with gameplay, art, audio, and UI

Milestone Stages

Prototype → Vertical Slice → Alpha → Beta → Launch

Scope Discipline

Defer non-critical features until slice is validated

8. Technical Art Workflows

Bridge art and engineering. Where performance, visuals, and scalability meet.

Shader Libraries

Shared, optimized shaders for consistency

LOD Pipelines

Automate creation and validation of LODs

Optimization Budgets

Polycount, texture resolution, draw calls

Asset Validation

Automated checks for pivots, scales, UVs, metrics
